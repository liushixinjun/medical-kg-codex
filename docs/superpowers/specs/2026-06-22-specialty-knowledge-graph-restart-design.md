# 专科知识图谱从零重建设计

日期：2026-06-22  
状态：待用户书面复核  
目标文件：`AI自动化工具-文献指南解析.md`、`专科知识图谱Schema标准.md`

## 1. 目标

从原始 PDF、DOCX、DOC、教材和经确认的专家 TXT 出发，按统一 Schema 生成可审计、可增量合并的专科知识图谱数据实例。新流程不读取、不继承、不修补任何历史解析文本、旧图谱数据或旧中间结果。

默认交付到“标准数据实例＋质量审核报告”；Neo4j 导入必须在数据实例验收后由用户单独确认。

## 2. 每次启动的人工确认闸门

每次新任务必须先展示并等待用户明确确认：

1. `scope_type`：`specialty`、`category` 或 `disease`。
2. `scope_target`：如“心血管内科”“心肌病”“肥厚型心肌病”。
3. 原始指南/教材根目录、候选文件数量和文件类型统计。

未同时确认执行范围和原始文献路径，不得扫描、解析、写文件或修改数据库。即使目录与上次相同，也必须重新确认。

## 3. Schema 架构

采用方案 B：统一核心 Schema＋专科/疾病大类配置＋单病种规则。

### 3.1 Schema 标准 V1.0 核心层

从零定义统一的临床实体、关系、编码、证据和闭环规则，不引用任何历史版本或迁移说明，不因单一专病增加全局字段。核心层同时定义从零重建、批次执行和受控合并所需的通用元数据：

- `batch_id`
- `scope_type`
- `scope_target`
- `source_type`
- `document_id`
- `segment_id`
- provenance 证据集合
- `merge_status`
- `source_version`
- `conflict_status`

### 3.2 专科/疾病大类配置层

配置某专科或疾病大类的必需、可选、不适用模块和术语。例如心肌病可配置遗传检测、心脏磁共振、家族筛查、分型和猝死风险评估。

配置只控制抽取和验收，不创建新的临床图谱层。

### 3.3 单病种规则层

疾病特异知识使用既有 `ClinicalRule`、`ThresholdRule`、适用条件和证据表达。例如 HCM 的 LVOT 梗阻阈值、STEMI 的再灌注时间窗。

只有概念被多个疾病稳定复用、现有 Schema 无法无损表达且具有诊疗推理价值时，才进入后续 Schema 升级评估。

## 4. 来源体系

来源优先级：

1. 最新权威指南、正式共识：正式诊断标准、治疗推荐、药物剂量、阈值、随访和证据等级。
2. 权威教材《内科学》：定义、病因、病理生理、流行病学、症状体征、基础诊断和通用治疗。
3. 可追溯的专家整理 TXT：补充指南和教材未覆盖的知识。
4. 用户人工下载整理的网页 TXT：仅作为待核验补充来源，不得自动抓取后直接入图。

教材不冒充指南分级：

```text
source_type=authoritative_textbook
source_authority=authoritative_textbook
knowledge_strength=high
clinical_applicability=general
recommendation_class=N/A
evidence_level=N/A
```

指南与教材冲突时，正式推荐、诊断阈值和证据等级以最新权威指南为准，并输出冲突台账，不静默覆盖。

## 5. 《内科学》基础证据库

《内科学》首次执行时完整解析一次，建立可复用的教材证据库：

- 逐页清点和文本质量审计。
- 保留篇、章、节、页码、段落和稳定 `segment_id`。
- 建立疾病名称、别名和章节索引。
- 不预先把全书内容写入所有疾病节点。

后续任务根据 `scope_type/scope_target` 只调用相关章节和证据分段，不重复解析整本书。教材证据库是可追溯来源库，不是 AI 无痕学习结果。

## 6. PDF 解析与质量闸门

PDF 处理分为两个独立指标：

- `page_accounting_rate=100%`：每页均有页码、页面分类和处理状态。
- `eligible_content_pass_rate=100%`：正文、表格、推荐、剂量、阈值、诊断标准和治疗路径等临床内容页必须解析合格或明确阻断。

目录、索引、版权和空白页可不抽取知识，但必须计入页面清点。原生文本不足时使用第二解析引擎和 OCR。仍无法可靠恢复的文献不得参与知识抽取。

所有新文本保存文档、页面、章节和分段边界。禁止使用文档数组顺序、全局滑动窗口或相邻文件位置绑定疾病。

## 7. 疾病绑定与抽取

抽取前先建立：

```text
(batch_id, document_id, segment_id, disease_code, pathway_element)
```

文献仅因参考文献、目录、缩写表、并发症或偶然提及某疾病，不得获得该疾病的全文所有权。多疾病指南必须在文档内部按章节和疾病锚点分段绑定。

临床实体或规则进入关系时，其规范名称或受控别名必须在同一证据分段中出现；definition 必须与目标疾病名称或别名相关。不同疾病出现相同定义或相同证据片段时必须阻断并复核。

## 8. 数据实例与闭环验收

每个任务独立输出版本化批次包，包括：

- 原始文献清单与去重结果
- 页面审计和阻断清单
- 干净文本与分段索引
- 实体、关系、Evidence 和 Guideline 实例
- 疾病诊疗路径覆盖矩阵
- 缺失原因和解决方案
- 冲突台账
- Schema、编码、证据和污染质量报告

诊疗路径按适用性配置验收：定义、病因/危险因素、病理生理、流行病学、症状、体征、检查、检验、指标/阈值、诊断、鉴别诊断、分型/分期、风险分层、治疗方案、药物、操作、适应证、禁忌证、并发症、预后和随访。

必需环节没有有效证据时必须报告缺口，不得用其他环节的证据数量弥补。

## 9. 批次合并

从零开始时建立空的标准主图谱。后续专科、疾病大类或单病种任务均先生成独立批次包，验收通过后才能合并。

合并规则：

- 节点按全局 `code`、规范名称和受控别名归一。
- 关系语义键为 `(source.code, relationType, target.code)`。
- 多来源证据聚合到同一语义关系，保留完整 provenance。
- 冲突进入冲突台账，不自动覆盖。
- 合并前生成主图谱快照；合并失败可回滚。
- 后续抽取不得读取历史解析文本，但合并阶段必须读取当前标准主图谱用于去重和一致性检查。

## 10. 两阶段交付

第一阶段默认输出标准数据实例和质量审核报告，不修改 Neo4j。

第二阶段仅在用户确认第一阶段验收通过后执行：生成导入包、备份当前数据库、导入 Neo4j、执行全库审计并输出验收报告。

## 11. 新版 Skill 结构

新版 `AI自动化工具-文献指南解析.md` 必须删除所有历史批次数字、旧错误案例、旧目录状态、修复过程和特定数据库现状，只保留可重复执行的正式规则：

1. 启动确认闸门
2. 范围解析
3. 来源清单和优先级
4. 《内科学》基础证据库
5. PDF/DOCX/DOC/TXT 解析
6. 页面和文本质量审计
7. Schema 标准 V1.0 映射
8. 疾病锚点和证据绑定
9. 数据实例生成
10. 闭环、污染、编码和证据质量闸门
11. 独立批次交付
12. 受控合并
13. 可选 Neo4j 导入

## 12. 完成标准

- Skill 不含历史数据、历史修复过程和当前数据库统计。
- Schema 标准 V1.0 只包含统一核心和必要通用元数据，不包含专病字段堆积或历史迁移内容。
- `AI自动化工具-文献指南解析.md` 内部版本固定从 `V1.0` 开始。
- Schema 固定文件名为 `专科知识图谱Schema标准.md`，内部版本固定从 `V1.0` 开始。
- 任意新任务必须在范围和路径确认前停止。
- 旧解析文本和旧图谱不得成为新批次的知识来源。
- 数据实例的每条核心知识均可追溯到原始来源分段。
- 独立批次可以无损、去重、可回滚地合并到标准主图谱。
