## 2026-07-12 09:29:41｜历史全书回捞候选池不能批量复用

### 典型表现

历史候选池中存在大量“看似可归并”的症状、体征、检查、治疗等实体，但证据片段常存在章节坐标不对齐、疾病名未出现、否定/禁忌上下文、泛化实体等问题。

### 根因

历史回捞候选是早期宽松抽取结果，不是当前 D6 骨架基线下的严格证据事实。它可以提升召回线索，但不能替代“当前骨架 + 原文证据 + G1/G2审计”的正式抽取流程。

### 防复发规则

1. 历史候选池最高用途是“重抽取线索库”，不得直接生成入库 delta。
2. 任意历史候选必须回到教材/PDF原文，重新抽取证据、实体、关系和槽位。
3. 只有当前流程重新生成并通过 G1/G2 审计的数据，才允许进入服务器导入。
4. 抽样核查报告中如 `direct_batch_reuse_allowed=false`，后续流程必须阻断批量复用。

## 2026-07-12 08:39:56｜不能把历史候选池问题当成当前骨架问题

### 典型表现

分析全书回捞候选时，发现疾病名映射、章节坐标和证据文本噪声问题，如果不区分产物时间，容易误判为“本周新骨架仍有严重问题”，造成无效返工。

### 根因

当前 D6 教材骨架是 2026-07-09 之后的新基线，而 `textbook_fullbook_backfill_index.csv` 是 2026-06-25 的历史候选池。两者不是同一阶段产物，不能用旧候选池质量反向否定当前骨架。

### 防复发规则

1. 每次质量审计必须先列出输入文件 mtime 和产物阶段。
2. 只有当前基线之后生成的正式骨架/入库产物，才计入当前质量问题。
3. 当前基线之前的旧候选池，只能标记为“历史候选池归并风险”，不得作为当前骨架扣分。
4. 如需复用历史候选，必须经过名称映射、证据文本质量、章节坐标口径三道审计，再进入抽样复核。

## 2026-07-12 08:04:56｜全书回捞候选不能直接归并到D6教材槽位

### 典型表现

全书回捞索引中已有大量症状、体征、检查、治疗等深层实体，但疾病名与 D6 教材章节名不完全一致，且 `line_number` 与 `docx_start_para/docx_end_para` 疑似不是同一坐标系。

### 根因

全书回捞、D6教材矩阵、图谱Disease节点、alias字典之间缺少统一疾病名称映射和统一章节坐标口径。

### 防复发规则

1. 未完成疾病名称标准化映射前，不得把全书回捞候选直接归并到疾病槽位。
2. 未确认 `line_number` 与 D6 段落坐标同源前，不得按段落范围自动判定候选归属。
3. 先输出名称对齐审计和疑似跨章节噪声清单，再进入 G1/G2 候选审计。

## 2026-07-11 23:55:00｜长文档膨胀导致上下文压缩后难接续

### 典型表现

1. 主 SKILL、步骤记录、踩坑日志越来越大，读取慢、token 消耗高。
2. 同一规则同时散落在 SKILL、步骤记录、踩坑日志、方案文件中，后续难判断哪个是最新。
3. 换账号或上下文压缩后，容易重新问、重新解释、重复犯错。

### 根因

把“执行总纲、历史过程、事故复盘、软件化索引”混在同一长文档中，没有分层。

### 防复发规则

1. 主 SKILL 只保留总纲和硬闸门。
2. 细节进入 `技能文档_skill_docs/` 附录。
3. 历史步骤和踩坑按月归档。
4. 可复发错误必须进入结构化错误指纹索引。
5. 每轮结束必须更新 `当前进度与后续计划_交接.md`。
