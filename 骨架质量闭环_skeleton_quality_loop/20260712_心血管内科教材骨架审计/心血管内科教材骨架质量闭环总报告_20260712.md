## 3. 本轮发现的关键问题

最关键问题不是“再抽一遍”，而是“归并前必须先对齐名称和坐标”。

全书回捞归并审计结果：

| 指标 | 数值 |
|---|---:|
| 全书回捞索引总行数 | 13,066 |
| 可映射到骨架槽位的行数 | 13,034 |
| 章节内段落对齐行数 | 20 |
| 章节外/疑似噪声行数 | 13,014 |
| 全书回捞疾病名数 | 68 |
| 未在D6矩阵命中的疾病名数 | 34 |

解释：

1. `textbook_fullbook_backfill_index.csv` 中的 `line_number` 与 D6 的 `docx_start_para/docx_end_para` 很可能不是同一坐标系，不能直接用来自动归并。
2. 全书回捞疾病名与 D6 教材章节名存在明显差异，例如部分专病、亚型、缩写未在 D6 主题名中直接命中。
3. 如果现在直接把 13,034 条深层实体并入疾病槽位，会有较高误归属风险。
