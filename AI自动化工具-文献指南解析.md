## 0.1 当前建设目标

2026 年 7 月心血管内科优先完成以下疾病大类/专病图谱，按“当前 D6 教材骨架基线 + 教材/指南原文重抽取 + G1/G2 审计 + 受控入库”推进：

```text
冠心病
心肌病
急性心肌梗死
心力衰竭
心律失常
起搏治疗相关疾病
瓣膜病

```

本月不再复用 2026-06-25 历史全书回捞候选池直接补骨架；旧候选池最高用途是“重抽取线索库”。

## 1. 核心原则

| 序号 | 原则 | 解释 |
|---:|---|---|
| 1 | 基础权威教材/专科专著搭骨架 | 先形成疾病概述、病因、机制、临床表现、检查、诊断、鉴别、分层、治疗、随访等基础框架。 |
| 2 | 指南/共识/路径填血肉 | 指南用于补推荐等级、证据等级、适用人群、禁忌、剂量、路径、随访和 CDSS 可执行规则。 |
| 3 | Schema 是硬边界 | 所有实体、关系、字段必须映射到 `专科知识图谱Schema标准.md`；Schema 表达不了时记录 gap，不硬塞。 |
| 4 | 证据链必须可追溯 | 推荐、诊断、阈值、禁忌、药物、手术等临床动作必须能追到 Evidence 或权威教材来源。 |
| 5 | 本地审计先于写库 | G0/G1/G2 未通过，不进入 Neo4j；不能直接把候选写入服务器。 |
| 6 | 错误复利 | 重复错误必须进入 `_全局复利与踩坑日志.md` 和 `项目管理中心_project_management/05_全局错误指纹索引_error_fingerprint.csv`。 |
| 7 | 当前基线优先 | 质量判断以当前 D6/服务器复核基线为准；历史候选池问题只算复用风险，不反向否定当前骨架。 |

## 2. 每批次启动前必须确认

| 字段 | 中文含义 | 是否必填 | 示例 |
|---|---|---:|---|
| `top_specialty` | 顶层学科 | 是 | 心血管内科 |
| `disease_category` | 疾病大类 | 是 | 冠心病 |
| `scope_type` | 执行范围类型 | 是 | 疾病大类 / 单病种 / 专科全量 |
| `scope_target` | 本批次目标 | 是 | 急性心肌梗死 |
| `source_roots` | 指南/文献来源路径 | 是 | `E:\...\心血管内科\诊疗指南` |
| `textbook_roots` | 教材/专著路径 | 是 | `E:\...\心血管内科\书籍教材` |
| `output_root` | 输出路径 | 是 | `E:\...\心血管内科文献集合` |
| `batch_id` | 批次编号 | 是 | `BATCH-CARD-CAD-20260711-001` |
| `neo4j_action` | 是否允许写库 | 是 | `none` / `import_after_audit` |

缺任一必填项，不开始抽取。

## 3. 标准执行流

| 阶段 | 名称 | 目标 | 通过条件 |
|---|---|---|---|
| G0 | 启动预检 | 确认范围、路径、来源、批次编号、Schema 版本 | 预检报告无阻断 |
| G1 | 文献解析与实体抽取 | 从教材和指南生成节点、关系、证据、来源清单 | 本地 JSONL/CSV 产物完整 |
| G2 | 本地质量审计 | 查空壳、重复、技术名、缺证据、required 缺口 | 硬闸门为 0 或明确 blocked |
| G3 | 受控入库 | 仅在 G2 通过且用户允许时写 Neo4j | 导入日志和回滚文件存在 |
| G4 | 服务器复核与交接 | 从服务器统计实际实体、关系、质量指标 | 复核报告、可导入文件、下一步建议齐全 |

## 4. 不同知识层的边界

| 层级 | 来源 | 用途 | 典型字段 |
|---|---|---|---|
| 教材骨架层 | 内科学、专科教材、权威专著 | 建立基础临床框架，不直接替代指南推荐等级 | `source_type=authoritative_textbook`、`knowledge_layer=skeleton` |
| 指南证据层 | 指南、共识、路径、标准 | 形成正式诊疗推荐、证据等级、禁忌、适用人群 | `source_type=guideline`、`recommendation_class`、`evidence_level` |
| CDSS 决策层 | 图谱 + 规则引擎 + 患者数据 | 根据触发条件动态推荐诊断、检查、治疗、随访 | `RecommendationStatement`、`ClinicalRule`、`TriggerCondition` |

## 5. 质量硬闸门

以下问题出现时，不得声明“可正式 CDSS 上线”：

1. 疾病 required 槽位缺失，且未标明 `SOURCE_DOES_NOT_COVER` 或 blocked 原因。
2. 诊断标准只有标题，没有 `has_diagnostic_component` 明细。
3. 鉴别诊断只有疾病名，没有鉴别要点、排除检查或判别规则。
4. 治疗方案没有下游具体药物、手术、操作、禁忌、适用条件或证据链。
5. 药物类别没有具体药物，或药品主名是英文缩写而未放入 alias。
6. 出现 `EXAM-TTE` 这类技术编码名作为医生可见名称。
7. 同类型同名实体重复，未归并 alias 或关系。
8. 推荐展示时只显示疾病级 Evidence 池，不能定位到当前推荐动作的主证据。
9. 未经 G2 审计直接写 Neo4j。
10. 任务结束未提供可导入文件、审计报告、服务器复核或下一步交接。
11. 使用历史候选池直接生成入库 delta，或把旧候选池抽样问题当作当前骨架缺陷。

## 6. Neo4j 写库规则

默认不写库。只有同时满足以下条件才允许 G3：

1. 用户明确允许导入或本批次 `neo4j_action=import_after_audit`。
2. G2 审计通过。
3. 存在导入包、回滚清单和 postcheck 脚本。
4. 写库后必须跑服务器硬闸门复核。

数据库连接信息从 `图谱数据库链接.txt` 或用户配置读取，不写入 Git，不写入日志明文密码。

## 9. 禁止事项

- 不允许因为单个非阻断节点拖住全局批次进度；应记录阻断并继续可并行部分。
- 不允许把候选、补丁、Trae 直接修改结果绕过证据回捞直接写 Neo4j。
- 不允许删除、移动、重命名旧批次目录，除非用户明确批准。
- 不允许新增纯英文文件夹名；新目录必须采用“中文说明_english_suffix”。
- 不允许在医生界面暴露技术编码名、内部节点 ID、全量 Evidence 池。
- 不允许用“抽检通过”代替硬闸门全量检查。
