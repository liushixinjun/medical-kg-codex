# 心血管内科教材骨架质量评估

评估时间：2026-07-07 23:35:00
评估对象：《内科学（第10版）》心血管内科章节；优先比对冠心病、心肌病、心力衰竭、心律失常四类。
评估目的：判断“内科学搭骨架，指南 PDF 填血肉”是否已经真实落地，并反查 Schema 是否足以承载教材骨架。

---

## 一、结论

当前方向是正确的：内科学教材适合做心血管专科基础骨架，指南/共识适合做精细化决策。

但当前图谱尚未真实达到“教材骨架完备”的标准，不能把现有骨架库视为已经完成。主要问题不是单纯节点数量，而是以下三类：

1. 教材章节原文中已经存在的基础内容，没有稳定按“概述—病因机制—临床表现—检查—诊断鉴别—分型/分层—治疗”拆成结构化知识。
2. 部分疾病 `description` 被错误章节污染，说明原抽取缺少“疾病章节锚点”和“章节槽位”校验。
3. Schema 基本能表达大部分临床内容，但缺少教材骨架专用的来源层、章节槽位、页码范围和定义/概述强校验规则。

因此下一步不应继续盲目扩新病种，而应先对四个重点大类做“教材骨架根治修复”，再用指南补决策。

## 三、四类章节结构对比

  - 病理生理
- 第二节 慢性心力衰竭

  - 辅助检查
  - 诊断与鉴别诊断

### 3.2 心律失常

  - 心律失常分类
  - 心律失常发生机制
  - 心律失常的诊断
- 第二节 窦性心律失常
- 第三节 房性心律失常
- 第四节 房室交界性心律失常
- 第五节 室性心律失常
- 第六节 心脏传导阻滞
- 第七节 抗心律失常药物的合理应用
- 第八节 心律失常的介入治疗和手术治疗

常见子结构：

- 病因
- 临床表现

## 四、统一教材骨架槽位

四类疾病可以统一为以下 10 个教材骨架槽位：

| 槽位 | 中文含义 | 是否普遍存在 | 当前 Schema 支持度 | 问题 |
|---|---|---|---|---|
| overview | 疾病概述/定义/临床意义 | 普遍存在 | 部分支持 | 目前主要靠 `Disease.description/definition`，缺少原文锚定强校验 |
| etiology | 病因/危险因素 | 普遍存在 | 支持 | 需区分病因和危险因素，不能混放 |
| pathogenesis | 发病机制/病理生理 | 普遍存在 | 支持 | 心肌病“病理改变”与“病理生理”需细分 |
| epidemiology | 流行病学 | 部分存在 | 支持 | HCM、心衰等应结构化为发病率/患病率/人群 |
| clinical_manifestation | 临床表现 | 普遍存在 | 支持 | 必须区分核心表现、并发症表现、鉴别对象表现 |
| exam_lab | 实验室和辅助检查 | 普遍存在 | 支持 | 心电图特征、影像指标、检验指标应拆到组件级 |
| diagnosis_differential | 诊断与鉴别诊断 | 普遍存在 | 支持但需强制拆解 | 不能只有“诊断标准”标题或鉴别疾病名 |
| classification_risk | 分型/分期/分级/危险分层 | 部分存在 | 支持 | 需区分疾病分型、严重度分级和风险分层 |
| treatment | 治疗 | 普遍存在 | 支持但需分层 | 教材治疗原则不等于指南推荐动作 |
| prognosis_followup_prevention | 预后/随访/预防 | 部分存在 | 支持 | 教材常有预防/预后，部分病种未必有随访 |

## 五、当前服务器图谱质量评估

### 5.1 结构数量不能代表质量

服务器中四类相关疾病均有大量关系，例如 HCM 有症状、体征、检查、治疗、诊断标准、证据等关系。但这只能说明“有连接”，不能说明“连接正确”。

旧审计能发现空壳、重复、非 KGNode、药物类别等技术问题，但没有发现“疾病章节绑定错误”和“description 污染”。

### 5.2 定义字段缺失

服务器抽查四类重点疾病：

| 类别 | 疾病数 | `definition` 为空 |
|---|---:|---:|
| 冠状动脉粥样硬化性心脏病 | 10 | 10 |
| 心肌病 | 12 | 12 |
| 心力衰竭 | 11 | 11 |
| 心律失常 | 29 | 29 |
| 心脏骤停与心脏性猝死 | 2 | 2 |

结论：虽然 `description` 有内容，但 Schema 已将 `definition` 作为 required，当前实现没有落实。

### 5.3 已发现的 description 污染样例

| 疾病 | 当前问题 |
|---|---|
| 不稳定型心绞痛 | `description` 命中“康复医学/呼吸康复”段落，不是 UA 定义 |
| 急性心肌梗死 | `description` 命中“窦性停搏或窦性静止”段落，不是 AMI 定义 |
| 急性冠脉综合征 | `description` 命中“急性心力衰竭”段落，不是 ACS 定义 |
| 缺血性心肌病 | `description` 命中“非扩张型左心室心肌病”段落，不是 ICM 定义 |
| 陈旧性心肌梗死 | `description` 命中“心脏骤停”段落，不是陈旧性 MI 定义 |
| 心房颤动 | `description` 命中 HCM/心室充盈受限机制段落，不是 AF 定义 |
| 右心衰竭 | `description` 命中限制型心肌病定义，不是右心衰竭定义 |

这属于 P0 质量问题。它说明抽取逻辑曾经用关键词命中替代章节锚定，导致跨章节污染。

### 5.4 HCM 教材骨架样例

PDF 第 306 页 / 书内第 275 页中，HCM 第一段至少应拆成：

| 原文信息 | 应进入图谱的位置 |
|---|---|
| HCM 是以左心室和/或右心室肥厚，伴舒张功能障碍为特征的心肌病 | `Disease.definition` |
| 需排除后负荷增加或系统性疾病导致的心肌肥厚 | `DiagnosisCriteria -> has_diagnostic_component -> ClinicalRule` 或鉴别/排除规则 |
| 成人发病率 2‰、儿童 0.02-0.05‰、患病率 0.29‰ | `Epidemiology` + 指标字段 |
| 临床表现差异大 | 概述/临床表现说明，不应当作单个症状 |
| 部分病人以 SCD 为首发表现 | `Prognosis/Complication/ClinicalRule`，并连接 SCD |
| 是青少年和运动猝死的重要原因 | `RiskFactor/PatientState/Prognosis` |
| 原文整段 | `Evidence.evidence_text`，带 PDF 页码、书内页码、章节路径 |

当前图谱只把短定义放入 `description`，没有写入 `definition`，也没有完整结构化发病率、患病率、SCD 首发表现和青少年/运动猝死风险。

## 六、Schema 骨架能力评估

### 6.1 Schema 已经能承载的部分

| 教材内容 | Schema 表达 |
|---|---|
| 症状/体征 | `Symptom`、`Sign` |
| 病因 | `Etiology` |
| 发病机制/病理生理 | `Pathophysiology` |
| 流行病学 | `Epidemiology` |
| 检查/检验/指标/阈值 | `Exam`、`LabTest`、`ExamIndicator`、`ThresholdRule` |
| 诊断标准 | `DiagnosisCriteria -> has_diagnostic_component` |
| 鉴别诊断 | `DifferentialDiagnosis`、`has_differential_point`、`requires_exclusion_exam`、`may_block_action` |
| 分型/分级 | `ClassificationStage` |
| 危险分层 | `RiskStratification`、`ScoringScale` |
| 治疗 | `TreatmentPlan`、`Medication`、`Procedure`、`Indication`、`Contraindication`、`TreatmentTiming` |
| CDSS 推荐 | `PathwayStage`、`ClinicalRule`、`RecommendationStatement` |

结论：Schema 主体方向正确，不需要推倒。

### 6.2 Schema 缺失或不够清楚的部分

#### 1. 缺少教材骨架槽位字段

当前没有强制字段标记一条节点/关系/证据属于教材的哪个槽位，例如：

- `overview`
- `etiology`
- `pathogenesis`
- `clinical_manifestation`
- `exam_lab`
- `diagnosis_differential`
- `classification_risk`
- `treatment`

影响：无法审计“某疾病教材章节是否完整抽取”，只能看节点数量。

建议：在节点、关系或 Evidence 上新增 `skeleton_slot` 字段，并把它纳入审计。

#### 2. 缺少知识层级字段

当前只有 `source_type=authoritative_textbook/guideline`，不足以区分：

- `textbook_core`：教材基础骨架
- `guideline_supplement`：指南补充知识
- `guideline_decision`：带推荐等级/证据等级的决策知识
- `screening_context`：筛查/共病/背景上下文
- `cross_reference`：章节引用

影响：前端容易把指南补充、筛查上下文、并发症表现都当作核心症状或核心治疗。

建议：新增 `knowledge_layer` 或 `clinical_use_layer` 字段。

#### 3. 来源页码字段不够

当前 `source_page` 只有一个值。PDF 校验发现标题和内容可跨页。

建议新增：

- `pdf_page_start`
- `pdf_page_end`
- `book_page_start`
- `book_page_end`
- `source_section_path`
- `text_anchor`

#### 4. definition 字段有要求但没有硬闸门

Schema 规定 `definition` required，但服务器四类重点疾病 `definition` 全为空。

建议：新增硬闸门：

```text
disease_definition_empty_count = 0
disease_definition_source_mismatch_count = 0
description_cross_chapter_pollution_count = 0
```

#### 5. 病理改变与病理生理需要细分

心肌病章节明确区分“病理改变”和“病理生理”。当前只有 `Pathophysiology`，容易混放。

建议二选一：

- 保守方案：`Pathophysiology` 增加 `mechanism_subtype=pathological_change/pathophysiology/mechanism`
- 严格方案：新增 `PathologyFinding` 实体，承载解剖、组织学、病理改变

建议先用保守方案，避免 Schema 膨胀。

#### 6. TreatmentPlan 层级需要明确

教材里的治疗往往是“治疗原则/治疗目标/药物类别/手术类别”，指南才提供具体推荐动作。

建议 `TreatmentPlan` 增加：

- `plan_level=principle/strategy/action_bundle/action`
- `source_role=textbook_framework/guideline_decision`

这样可避免把“治疗原则”错误当成可直接执行医嘱。

## 七、内科学与指南 PDF 的边界

### 7.1 教材做什么

教材负责搭骨架：

- 疾病定义
- 疾病分类
- 病因和发病机制
- 基础临床表现
- 基础检查与诊断路径
- 鉴别诊断框架
- 基础治疗原则
- 预后、预防、随访框架

教材证据强度高，但不等同于指南推荐等级。教材来源的 `recommendation_class/evidence_level` 应保持 `N/A` 或“教材证据”。

### 7.2 指南做什么

指南负责填血肉：

- 精确适应证
- 禁忌证/排除条件
- 推荐等级
- 证据等级
- 时间窗
- 剂量/疗程
- 决策优先级
- 特殊人群
- 质控指标
- 随访频率

### 7.3 冲突处理

同一临床场景下：

1. 决策推荐以最新、权威、适用地区的指南为主。
2. 教材保留为基础知识和背景证据。
3. 教材与指南不一致时，不删除教材证据，但不得用教材覆盖指南推荐等级。

## 八、下一步完善方案

### 阶段 1：先补标准，不直接改库

修改 Schema 和 SKILL，新增以下硬规则：

1. 教材抽取必须输出 `textbook_skeleton_matrix.csv`。
2. 每条教材关系必须有 `skeleton_slot`。
3. 每条教材关系必须有 `knowledge_layer=textbook_core/cross_reference/screening_context`。
4. 疾病 `definition` 必须非空，且必须来自目标疾病章节定义段。
5. `description` 不得跨章节污染；若目标疾病名称不在来源章节路径或邻近标题中，阻断。
6. PDF/DOCX 必须双源校验：DOCX 抽结构，PDF 校页码。

### 阶段 2：四类优先重建教材骨架

优先顺序：

1. 冠心病
2. 心肌病
3. 心力衰竭
4. 心律失常

说明：这四类只是优先验证范围，不是最终范围。四类跑通并通过硬闸门后，必须按同一规则自动扩大到《内科学》第10版心血管内科其他疾病大类和疾病，包括高血压、心脏瓣膜病、心包疾病、感染性心内膜炎、先天性心血管病、主动脉和周围血管病、心血管神经症、肿瘤心脏病学及其他心血管疾病。

每类输出：

- 教材章节目录
- 疾病清单
- 每个疾病的 10 槽位覆盖矩阵
- P0 污染清单
- 修复 delta JSONL
- 本地审计
- 服务器导入前后对比

### 阶段 3：先修 P0，再补深层

P0 必须先修：

1. 疾病 `definition` 全量补齐。
2. 错误 `description` 全量清理。
3. 章节锚点错误关系删除或迁移。
4. HCM、AMI、ACS、UA、AF、右心衰竭等已发现污染样例先修。

P1 再补：

1. 症状体征按 core/complication/differential/screening 分层。
2. 诊断标准明细化。
3. 鉴别诊断下级要点、排除检查、治疗阻断。
4. 治疗原则与指南推荐动作分层。

### 阶段 4：再接指南决策层

教材骨架通过后，再接指南 PDF：

- 指南只补决策细节，不覆盖教材骨架原始定义。
- CDSS 推荐必须走 `RecommendationStatement`。
- 前端默认展示“教材核心骨架 + 当前阶段指南推荐”，不要展示疾病证据池。

## 十、最终判断

你的判断是对的：内科学教材基本覆盖临床医生看病的基础框架，适合作为专科图谱的骨架；指南 PDF 应作为精细化补充和决策依据。

但当前实现没有把这个思想完全落地。下一步必须把抽取方式从“关键词命中 + 关系堆叠”改成“章节锚定 + 骨架槽位 + 原文页码 + 结构化拆解 + 指南决策补充”。
