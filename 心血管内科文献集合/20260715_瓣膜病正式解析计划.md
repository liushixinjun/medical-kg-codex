# 瓣膜病正式解析完整执行计划（2026-07-15）

## 1. 批次目标

本批次从“教材骨架 + 指南/PDF解析”两个维度正式建设瓣膜病图谱，不再把 G0 预检或大类基线验收当成解析完成。

目标输出可以支持专科 CDSS 的瓣膜病诊疗流程，包括：

- 疾病定义和分类
- 病因和病理生理
- 症状、体征、检查、检验
- 诊断标准和鉴别诊断
- 严重程度评估和风险分层
- 药物、手术、介入治疗方案
- 禁忌证、排除条件、并发症处理
- 随访、预后、患者教育
- 每条关键推荐绑定来源、页码/段落、推荐等级、证据等级和原文摘要

## 4. 资料使用原则

| 资料类型 | 使用方式 |
|---|---|
| 《内科学》 | 搭基础骨架。若基础槽位缺失，先判定为抽取不充分，必须回原文复抽 |
| 国内/国际指南 | 填诊断、严重程度、治疗、介入/手术适应证、禁忌、随访、推荐等级和证据等级 |
| 护理/宣教文档 | 只补患者教育、术后随访、注意事项，不作为核心治疗推荐等级证据 |
| 非瓣膜性房颤资料 | 不作为瓣膜病核心资料；只有抗凝边界或排除条件明确相关时才引用 |
| 感染性心内膜炎资料 | 仅用于人工瓣膜心内膜炎、瓣膜手术并发症等场景 |

## 5. 一次性执行步骤

| 阶段 | 动作 | 产物 | 阻断条件 |
|---|---|---|---|
| G0 | 固化批次配置和来源清单 | batch_config、source_documents_manifest、去重清单 | 配置缺字段、路径不存在、纳入资料为 0 |
| G0.5 | 文档解析可行性检查 | PDF/DOCX 页级审计、OCR 阻断清单、清洗文本 | 核心指南无法解析且无 OCR 兜底 |

| G1-指南 | 从指南/PDF抽取证据 | guideline_evidence_index、证据摘要、来源页码 | 核心诊断/治疗证据缺失 |

| G1-质量审计 | 审计 required 槽位、同义词、重复、证据链 | G1 审计矩阵、缺口清单、可修复清单 | required 缺口未解释 |
| G2-入库前审计 | 确认是否可写库 | 入库包、回滚清单、阻断清单 | 证据链、诊断明细、治疗下游缺失 |
| G3-入库 | 受控写 Neo4j | Neo4j 导入日志 | 未通过 G2 不执行 |
| G4-服务器复测 | 服务器硬闸门和 CDSS 查询验证 | postcheck 报告 | 非KGNode、空壳、重复、证据范围串台 |
| 交付 | 输出统计和交接 | 交付报告、来源清单、可导入图谱文件清单、步骤记录 | 交接文件缺失 |

## 6. 本轮质量闸门

1. 教材骨架缺口不得直接说资料不足，必须先回《内科学》原文复抽。
2. 诊断标准必须有明细组件。
3. 鉴别诊断必须有对象、规则、排除依据。
4. 治疗方案必须能下钻到药物、手术/操作、禁忌或具体推荐动作。
5. 药物、检查、手术必须使用标准中文主名，缩写放入别名。
6. 推荐证据必须按当前疾病范围过滤，不能串到其他疾病。
7. 护理宣教不能替代指南推荐等级。
8. 未通过 G2 不写 Neo4j。
