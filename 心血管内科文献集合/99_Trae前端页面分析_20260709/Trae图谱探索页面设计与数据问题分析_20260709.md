# Trae 图谱探索页面设计与数据问题分析

分析对象：`http://192.168.3.27:4001/explore.html`

分析时间：2026-07-09

分析范围：页面设计、前端渲染逻辑、API 返回结构、Neo4j 图谱数据映射问题。

## 1. 总体判断

当前页面已经开始接入 `RecommendationStatement` 推荐卡片，但主体仍是旧的“疾病静态知识清单”页面，不是完整的“专病 CDSS 诊疗流程页面”。

核心问题不是图谱数据完全缺失，而是：

1. 页面默认展示逻辑仍按疾病维度一次性展示症状、检查、药物、证据池。
2. 新 CDSS 决策层与旧静态路径/诊断标准迁移结果混在一起。
3. 后端 API 没有完整映射 Neo4j 中已经存在的路径、阶段、规则、主证据字段。
4. 医生视角仍无法稳定回答：“当前推荐为什么触发、推荐什么、依据哪条指南哪一页”。

## 2. 已确认的页面结构

页面导航包含：

```text
数据总览
图谱探索
网络探索
数据覆盖分析
临床诊断模拟
路径编辑
临床审核
图谱数据字典
Schema标准
指南库
医学术语库
```

`explore.html` 内部当前有三个视角：

```text
疾病视角
关系视角
实体视角
```

疾病视角当前按以下静态维度展示：

```text
临床表现：Symptom、Sign
诊断检查：Exam、LabTest
治疗方案：TreatmentPlan、Medication、Procedure
预后与随访：Prognosis、FollowUp、Complication
病因与机制：Etiology、Pathophysiology、Epidemiology
风险与诊断：RiskFactor、RiskStratification、DiagnosisCriteria、DifferentialDiagnosis
证据链与指南：Evidence、Guideline、ThresholdRule、ExamIndicator
```

这适合“知识图谱浏览”，不适合直接作为“专科 CDSS 推荐页面”。

## 3. 关键数据问题

### 3.1 AMI 动态 CDSS 推荐已存在，但 API 返回字段不完整

接口：

```text
/api/kg/disease/DIS-CARD-CAD-AMI/recommendations
```

返回 8 条推荐，其中 3 条为动态 CDSS 路径推荐：

```text
REC-CDSS-CAD-AMI-01-01-DIAG
REC-CDSS-CAD-AMI-02-01-REPERFUSION
REC-CDSS-CAD-AMI-03-01-PREVENT
```

但这 3 条在列表接口中出现：

```text
pathway_name 为空
stage_name 为空
rule_name 为空
primary_guideline_name 为空
```

这会导致前端无法按“路径 -> 阶段 -> 规则 -> 推荐”展示。

### 3.2 Neo4j 数据本身有这些字段，属于 API 映射问题

以 `REC-CDSS-CAD-AMI-02-01-REPERFUSION` 为例，Neo4j 可查到：

```text
pathway_name = 急性心肌梗死专病动态CDSS路径
stage_name = 急性心肌梗死再灌注决策阶段
rule_name = AMI再灌注/PCI评估规则
primary_source_name = STEMI CN 2019.pdf
primary_source_page = 4
primary_evidence_summary = AMI患者应结合分型、时间窗和PCI可及性评估急诊PCI、溶栓或冠脉造影。
recommendation_class = Ⅰ
evidence_level = A
```

所以这不是图谱数据缺失，是后端接口没有按最新 `RecommendationStatement` 模型映射字段。

### 3.3 详情接口也有字段映射缺口

接口：

```text
/api/cdss/recommendation/REC-CDSS-CAD-AMI-02-01-REPERFUSION
```

返回中存在：

```text
actions
evidence
guidelines
primary_guideline
```

但同时出现：

```text
stage_name = 空
pathway_name = 空
primary_guideline_name = 空
primary_evidence_code = 空
evidence.source_text = 空
```

建议接口直接映射：

```text
RecommendationStatement.primary_source_name -> primary_guideline_name
RecommendationStatement.primary_source_page -> primary_source_page
RecommendationStatement.primary_evidence_summary -> primary_evidence_summary
RecommendationStatement.primary_evidence_raw_excerpt -> primary_evidence_raw_excerpt
PathwayStage.name -> stage_name
ClinicalPathway.name -> pathway_name
ClinicalRule.name -> rule_name
```

### 3.4 推荐列表接口只暴露单个 action，不适合 CDSS 推荐卡片

AMI 再灌注推荐实际下游动作包括：

```text
再灌注治疗
冠状动脉造影
溶栓治疗
经皮冠状动脉介入治疗
```

但列表接口目前只有：

```text
action_code
action_name
action_entity_type
```

建议改为：

```json
{
  "actions": [
    {"code":"...", "name":"...", "entityType":"TreatmentPlan"},
    {"code":"...", "name":"...", "entityType":"Procedure"}
  ],
  "blocks": []
}
```

列表卡片可只展示前 2-3 个动作，详情再展开全部。

### 3.5 新旧路径混杂，前端未区分

AMI 的 `pathways` 中同时存在：

```text
PATHWAY-CDSS-CAD-AMI：急性心肌梗死专病动态CDSS路径
PATHWAY-CARD-CEAE0C4E99B9：急性心肌梗死治疗路径
```

前者是新 CDSS 动态路径；后者是历史静态路径迁移结果。

页面目前没有区分 `dynamic_cdss` 与 `legacy_migrated_static`，医生会误以为两个路径同等可用。

建议：

```text
默认只展示 dynamic_cdss 路径。
legacy_migrated_static 放到“历史知识/旧路径参考”折叠区。
```

### 3.6 疾病左侧树的完整率不是 CDSS 就绪率

当前左侧疾病树显示完整率，例如 AMI 100%。该完整率来自 17 个知识维度覆盖情况，不代表 CDSS 决策层完整。

建议同时展示两个指标：

```text
知识覆盖率：症状、体征、检查、治疗、证据等维度是否有数据。
CDSS就绪率：是否有 ClinicalPathway、PathwayStage、ClinicalRule、RecommendationStatement、Action、Evidence、Guideline。
```

否则医生和开发同事会误解“100%”代表可以正式上线。

## 4. 页面设计问题

### 4.1 疾病视角不应作为 CDSS 默认页

当前疾病视角展示方式是：

```text
疾病 -> 所有症状/体征/检查/药物/证据
```

这会回到旧 CDSS 的“一次性无脑圈出来”模式。

建议新增并默认进入：

```text
CDSS流程视角
```

结构：

```text
疾病
  -> 诊断确认阶段
  -> 风险分层阶段
  -> 治疗决策阶段
  -> 随访管理阶段
```

每个阶段下再展示规则卡和推荐卡。

### 4.2 证据区不应默认展示疾病级证据池

当前 AMI 页面“证据链与指南”默认列出 30 条证据和 27 个指南。

医生真正需要的是：

```text
当前推荐卡片 -> 主证据 -> 指南名称、页码、推荐等级、证据等级、原文摘要
```

建议疾病级证据池只作为“知识追溯”折叠区，不作为主流程展示。

### 4.3 首屏加载状态需要改进

新开页面时曾出现左侧疾病树为空、中央提示“请先从左侧选择一种疾病”的状态，但控制台无明显报错。

建议：

```text
加载中：显示“正在加载疾病列表...”
加载失败：显示 API 地址、HTTP 状态和重试按钮
加载成功但未选中：自动选择默认疾病，或提示选择
```

不能让用户看到空白树而不知道是没数据、加载慢还是接口失败。


默认进入 CDSS 流程视角。

### P0：推荐卡片以 RecommendationStatement 为根

卡片字段：

```text
推荐陈述

阶段

```

## 7. 是否属于图谱数据问题

本轮结论：

```text
主要不是图谱数据缺失。
主要是前端/API 没有按新版 CDSS 决策层模型取数和展示。
```

已确认 Neo4j 中存在动态推荐所需的路径、阶段、规则、主证据字段。

但仍有两类数据层后续可优化：

1. `Evidence` 节点自身的 `source_text/raw_excerpt` 为空时，前端应优先展示 `RecommendationStatement.primary_evidence_raw_excerpt`。
2. 旧迁移推荐中的 `recommendation_class/evidence_level=未结构化`，后续可逐步改成更友好的“未分级/指南未显式标注”。

## 8. 给 Trae 的一句话任务

请把 `explore.html` 从“疾病静态实体浏览页”升级为“专病 CDSS 流程页”：默认按 `Disease -> ClinicalPathway -> PathwayStage -> ClinicalRule -> RecommendationStatement -> actions/blocks/evidence/guidelines` 查询和展示；修复 API 字段映射，尤其是动态 CDSS 推荐的 pathway/stage/rule/primary_source/evidence 摘要字段；疾病级证据池只作为折叠参考，不作为推荐依据展示。
