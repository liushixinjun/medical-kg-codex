# 附录A：批次启动与范围确认

更新时间：2026-07-12 09:45:00

## 1. 目的

保证每次从 0 开始、新疾病大类、单病种增量、跨学科建设时，输入范围清楚，避免后续图谱合并混乱。

## 2. 必填启动信息

| 字段 | 中文含义 | 示例 | 缺失处理 |
|---|---|---|---|
| `top_specialty` | 顶层学科 | 心血管内科 | 阻断 |
| `disease_category` | 疾病大类 | 冠心病 | 阻断 |
| `scope_type` | 范围类型 | 疾病大类 | 阻断 |
| `scope_target` | 目标范围 | 急性心肌梗死 | 阻断 |
| `source_roots` | 指南 PDF 路径 | `...\诊疗指南` | 阻断 |
| `textbook_roots` | 教材路径 | `...\书籍教材` | 阻断 |
| `output_root` | 输出路径 | `...\心血管内科文献集合` | 阻断 |
| `batch_id` | 批次编号 | `BATCH-CARD-CAD-20260711-001` | 阻断 |

## 3. 范围类型

| 类型 | 说明 | 例子 |
|---|---|---|
| 专科全量 | 建设一个顶层学科骨架 | 心血管内科全量骨架 |
| 疾病大类 | 建设一个大类及其子类 | 冠心病、心肌病、心律失常 |
| 单病种 | 只做一个疾病 | 急性心肌梗死、肥厚型心肌病 |
| 增量修复 | 对既有图谱补槽位、证据链或规则 | 修复 AMI 诊断标准组件 |

## 3.1 心血管内科 2026-07 优先顺序

本月目标是完成心血管内科重点疾病大类/专病图谱。执行顺序固定如下，除非用户明确调整：

| 顺序 | 疾病大类/专病 | 执行口径 |
|---:|---|---|
| 1 | 冠心病 | 疾病大类，含 ACS、AMI、STEMI、NSTEMI、慢性冠脉疾病 |
| 2 | 心肌病 | 疾病大类，含 HCM、DCM、RCM、致心律失常性心肌病等 |
| 3 | 急性心肌梗死 | 单病种/专病样板，重点补诊断、再灌注、用药、随访证据链 |
| 4 | 心力衰竭 | 疾病大类，重点补分型、分期、用药、器械、随访 |
| 5 | 心律失常 | 疾病大类，含房颤、室上速、室性心律失常、缓慢性心律失常等 |
| 6 | 起搏治疗相关疾病 | 专题/治疗相关疾病，重点补适应证、禁忌、术后随访 |
| 7 | 瓣膜病 | 疾病大类，重点补狭窄/反流、手术/介入适应证 |
| 8 | 肺动脉高压 | 疾病大类/专病，重点补分型、筛查、风险分层、靶向治疗 |

每个批次开始前仍必须重新确认：顶层学科、疾病大类、目标范围、指南路径、教材路径、输出路径和是否允许写库。

## 4. 输出目录规则

新目录必须中文可读并带英文辅助，例如：

```text
项目管理中心_project_management
技能文档_skill_docs
日志归档_log_archive
治理脚本_governance_scripts
```

`心血管内科文献集合/` 暂按历史目录保留，不移动、不重命名、不拆分。
