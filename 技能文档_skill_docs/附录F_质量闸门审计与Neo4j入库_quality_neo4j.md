# 附录F：质量闸门、审计与Neo4j入库

更新时间：2026-07-12 09:45:00

## 1. G1 本地审计

检查本地节点、关系、证据、来源清单和导入包是否完整。

## 2. G2 硬闸门

| 闸门 | 通过要求 |
|---|---|
| 非 KGNode | 0 |
| 空壳节点 | 0 |
| 技术编码名 | 0 |
| 同类型同名重复 | 0 或明确合并策略 |
| 诊断标准无明细 | 0 |
| 鉴别诊断无下级内容 | 0 |
| 治疗方案无下游动作 | 0 |
| 药物类别无具体药物 | 0 |
| 推荐无证据链 | 0 |
| 历史候选池直接入库 | 0 |
| 未区分当前基线/历史候选池 | 0 |

## 3. G3 写库条件

默认不写库。允许写库时必须有：

1. 用户授权。
2. G2 通过。
3. 导入脚本、导入数据、回滚策略。
4. 服务器 postcheck。

历史候选池、Trae 候选、人工临时补丁不得直接进入 G3。必须先回到教材/指南原文，生成当前批次本地 JSONL/CSV，并通过 G2。

## 4. 服务器 postcheck

服务器统计必须从 Neo4j 实际查询，不用本地文件替代。输出要讲人话：

```text
本次入库后，服务器中疾病、实体、关系各增加多少；硬闸门是否仍为 0；哪些数据不能进入正式 CDSS。
```

## 5. 旧候选池复用闸门

旧候选池只能进入以下两类结果：

| 结果 | 后续动作 |
|---|---|
| 重抽取线索 | 回到当前教材/PDF原文重新抽取 |
| 不建议复用 | 不进入后续归并流程 |

禁止输出“可直接入库”“可直接生成 delta”。若确需复用单条候选，必须附当前原文证据、来源页码/段落、实体槽位和审计结果。
