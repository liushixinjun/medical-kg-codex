当前统一入口已接入主数据质量闸门。阻断项不为 0 时，脚本默认返回非 0 退出码，表示不得转正式 CDSS、不得继续叠加新病种。

旧的零散 postcheck 脚本仅作为历史排查资料，不再作为新批次默认入口。

## 2026-07-14 15:27 新增｜主数据质量闸门

后续新病种、旧批次精修和服务器入库后复核，必须执行主数据质量闸门：

```powershell
D:\Program Files Ai\python-venvs\medical-kg\Scripts\python.exe `
  公共执行层_kg_pipeline\主数据质量闸门_master_data_gate.py `
  --connection-file 图谱数据库链接.txt `
  --output-dir 心血管内科文献集合\00_全局质量体检_global_quality_audit\YYYYMMDD_主数据质量闸门 `
  --fail-on-blocking
```

必须为 0 的检查项：

| 检查项 | 必须为 0 的原因 |
|---|---|
| 同病种同类型同名重复直连 | 避免前端重复展示和 CDSS 推理入口不稳定 |
| 诊断标准无明细 | 诊断标准必须能下钻到具体条件 |
| 孤儿诊断标准 | 零关系节点不得进入正式图谱 |
| 治疗方案无下游 | 治疗方案必须能下钻到药物、操作、子方案、证据或路径动作 |
| 已替换重复节点仍被引用 | 已归并的重复节点不得继续被疾病、路径或规则引用 |

术语说明：标准主节点 = 同一临床概念最终保留和复用的主节点；重复节点的关系必须迁移到标准主节点。

## 1. G1 本地审计

检查本地节点、关系、证据、来源清单和导入包是否完整。

## 2. G2 硬闸门

| 闸门 | 通过要求 |
|---|---|
| 非 KGNode | 0 |
| 空壳节点 | 0 |
| 技术编码名 | 0 |
| 同类型同名重复 | 0 或明确合并策略 |
| 诊断标准无明细 | 0 |
| 鉴别诊断无下级内容 | 0 |
| 治疗方案无下游动作 | 0 |
| 药物类别无具体药物 | 0 |
| 推荐无证据链 | 0 |
| 历史候选池直接入库 | 0 |
| 未区分当前基线/历史候选池 | 0 |

## 3. G3 写库条件

默认不写库。允许写库时必须有：

1. 用户授权。
2. G2 通过。
3. 导入脚本、导入数据、回滚策略。
4. 服务器 postcheck。

历史候选池、Trae 候选、人工临时补丁不得直接进入 G3。必须先回到教材/指南原文，生成当前批次本地 JSONL/CSV，并通过 G2。

## 4. 服务器 postcheck

服务器统计必须从 Neo4j 实际查询，不用本地文件替代。输出要讲人话：

```text
本次入库后，服务器中疾病、实体、关系各增加多少；硬闸门是否仍为 0；哪些数据不能进入正式 CDSS。
```

## 5. 旧候选池复用闸门

旧候选池只能进入以下两类结果：

| 结果 | 后续动作 |
|---|---|
| 重抽取线索 | 回到当前教材/PDF原文重新抽取 |
| 不建议复用 | 不进入后续归并流程 |

禁止输出“可直接入库”“可直接生成 delta”。若确需复用单条候选，必须附当前原文证据、来源页码/段落、实体槽位和审计结果。
