# 附录G：日志归档、错误指纹与交接

更新时间：2026-07-12 09:45:00

## 1. 为什么拆分

上下文经常压缩，长日志会导致读取慢、token 消耗高、关键规则淹没。主日志只保留最近进度、索引和入口，历史按月归档。

## 2. 三类记录

| 文件 | 作用 |
|---|---|
| `当前进度与后续计划_交接.md` | 换账号或压缩上下文后的第一入口 |
| `AI自动化工具-文献指南解析步骤记录.md` | 每轮任务执行过程摘要 |
| `_全局复利与踩坑日志.md` | 可复发错误、根因和防复发规则 |

## 3. 错误指纹

可重复出现的问题必须登记到：

```text
项目管理中心_project_management/05_全局错误指纹索引_error_fingerprint.csv
```

登记字段至少包含：错误类型、典型表现、根因、检测方法、防复发规则、是否已有自动化测试。

## 4. 任务结束交接模板

```text
当前阶段：
已完成：
未完成/阻断：
是否写库：
关键文件：
下一步唯一建议：
风险提醒：
```

## 5. 月度目标交接

涉及阶段性建设目标时，交接文件必须写清：

```text
本月目标病种：
当前已完成：
当前不可复用的数据：
下一批次唯一建议：
是否允许写库：
```

本月心血管内科目标以 `项目管理中心_project_management/14_心血管内科月度建设计划_202607.md` 为准。
