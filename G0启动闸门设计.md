# G0 启动闸门设计

更新时间：2026-07-09

## 1. 目标

G0 启动闸门解决一个问题：任何新病种、新疾病大类、补丁修复、批量解析任务，在真正读取 PDF、生成图谱、写 delta 或连接 Neo4j 前，必须先确认范围、资料、骨架、权限和输出路径。

G0 不是解析脚本，也不是导入脚本；它是任务启动前的只读校验。

## 2. G0 的输入

唯一输入应为 `batch_config.yaml`。

最低必填字段：

```text
batch.batch_id
scope.top_specialty
scope.disease_category
scope.scope_type
scope.scope_target
source_roots.guideline_roots
source_roots.textbook_roots
output.output_root
execution_permissions.allow_neo4j_read
execution_permissions.allow_neo4j_write
execution_permissions.allow_skill_schema_change
```

## 3. G0 的硬校验

### 3.1 范围校验

必须明确：

```text
顶层学科
疾病大类
本批次疾病/分型/临床状态
本批次不处理的范围
是否为新批次
是否允许合并既有图谱
```

若 `scope_target` 为空，阻断。

若 `scope_type` 是“疾病大类”，必须允许多个专病；若是“专病”，必须列出具体病名。

### 3.2 路径校验

必须确认：

```text
指南路径存在
教材路径存在
输出根目录存在
术语字典路径存在或明确本批次不使用
外部权威路径存在或明确本批次不使用
```

路径不存在，阻断。

### 3.3 骨架校验

新病种启动前必须先回答：

```text
该疾病是否已在骨架库中
它属于疾病、分型、临床状态、并发症还是疾病大类
是否有 definition
是否有教材章节锚点
是否有基础槽位
是否存在同名/近义实体
是否已有 alias
```

骨架不满足时，不应直接进入指南解析；应先进入“骨架补齐任务”。

### 3.4 权限校验

G0 默认策略：

```text
allow_neo4j_read=false
allow_neo4j_write=false
allow_file_overwrite=false
allow_run_import_scripts=false
allow_run_repair_scripts=false
allow_run_delete_scripts=false
```

如果本批次需要 Neo4j，只允许明确写入：

```text
只读体检：allow_neo4j_read=true, allow_neo4j_write=false
正式导入：allow_neo4j_write=true 且 G2/G3 必须启用
```

任何 delete/dedupe/import/repair 脚本不得在 G0 阶段执行。

### 3.5 资料策略校验

必须明确：

```text
教材用于搭骨架
指南用于补推荐、证据、阈值、禁忌、路径
外部权威用于补教材/指南缺失的基础定义或罕见病知识
百度健康医典不得作为正式推荐证据
```

### 3.6 输出校验

G0 必须生成或确认批次输出目录，但不得覆盖历史目录。

建议输出目录命名：

```text
BATCH-学科缩写-疾病缩写-YYYYMMDD-序号_中文病种_EnglishName
```

若目录已存在，必须进入人工确认，不自动覆盖。

## 4. G0 的输出

G0 通过后应生成：

```text
00_scope_and_config/batch_config.yaml
00_scope_and_config/G0_startup_gate_result.json
00_scope_and_config/G0_startup_gate_checklist.csv
```

G0 阻断时应生成：

```text
G0_startup_gate_blockers.csv
```

阻断项必须写清：

```text
blocker_type
blocker_message
建议修复动作
是否允许继续
```

## 5. G0 通过标准

只有同时满足以下条件，才允许进入 G1 本地构建：

```text
范围明确
路径存在
骨架状态可接受
权限明确
输出目录安全
资料角色明确
禁止事项未被触发
```

## 6. 当前建议

下一步可以实现一个只读 `G0_startup_gate` 小工具，但不要迁移旧脚本，不要连接 Neo4j，不要写库。

第一版只需要：

```text
读取 batch_config.yaml
校验必填字段
校验路径存在
校验权限组合
输出 G0_startup_gate_result.json
```
