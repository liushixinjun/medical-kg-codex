# Schema 字段统一规则

版本：2026-07-11  
适用 Schema：`专科知识图谱Schema标准.md` V1.12  
本文件用途：解释 V1.12 后字段如何统一，给后续 dry-run 和脚本改造使用。  
本次动作：只定义规则，不写 Neo4j。

## 1. 核心原则

字段统一不是为了“好看”，而是为了让前端、审计脚本、CDSS 规则引擎都按同一口径查询。

必须统一的三类字段：

```text
1. 来源定位字段：这条知识从哪里来。
2. 教材骨架字段：这条知识属于教材哪一栏。
3. CDSS使用层级字段：这条知识能不能进入推荐。
```

## 2. skeleton_slot

`skeleton_slot` 表示“这条知识属于教材章节里的哪一栏”。

标准值：

| 标准值 | 中文含义 |
|---|---|
| `overview` | 疾病概述/定义 |
| `etiology` | 病因 |
| `pathogenesis` | 发病机制/病理生理 |
| `epidemiology` | 流行病学 |
| `clinical_manifestation` | 临床表现 |
| `exam_lab` | 检查/检验 |
| `diagnosis_differential` | 诊断与鉴别诊断 |
| `classification_risk` | 分型/分级/危险分层 |
| `treatment` | 治疗 |
| `prognosis_followup_prevention` | 预后/随访/预防 |

规则：

1. 教材骨架节点和关系都应具备 `skeleton_slot`。
2. `Disease.definition` 可继续保留 `definition_skeleton_slot`，但不能替代通用 `skeleton_slot`。
3. 同一实体被多个疾病复用时，以关系上的 `skeleton_slot` 为准。

## 3. knowledge_layer

`knowledge_layer` 表示“这条知识在 CDSS 中怎么用”。

标准值：

| 标准值 | 中文含义 | 使用规则 |
|---|---|---|
| `textbook_core` | 教材基础骨架 | 基础定义、病因、机制、临床表现、基础诊疗原则 |
| `guideline_supplement` | 指南补充知识 | 补教材没有的检查、特殊人群、治疗细节 |
| `guideline_decision` | 指南决策知识 | 可进入 `RecommendationStatement` 推荐卡片 |
| `screening_context` | 筛查/背景上下文 | 只能背景展示，不可作为核心诊断或推荐 |
| `cross_reference` | 跨章节引用 | 只能说明被其他章节提到，不可写入核心画像 |

历史值处理：

| 历史值 | 标准处理 |
|---|---|
| `textbook_skeleton` | 迁移为 `textbook_core` |
| `evidence` | 不直接保留；按 `source_type` 和用途拆分 |

## 4. source_section_path

`source_section_path` 是完整章节路径。

示例：

```text
第三篇 循环系统疾病 > 第六章 心肌疾病 > 第一节 肥厚型心肌病 > 诊断与鉴别诊断
```

规则：

1. 教材、指南、长 PDF 必须保存完整章节路径。
2. 不能只保存“治疗”“诊断”等短标题。
3. 章节路径用于防止跨章节误抓，例如把肺栓塞章节里的“心肌梗死鉴别”错写成心肌梗死核心知识。

## 5. Evidence 与 RecommendationStatement 字段边界

| 字段所在实体 | 保存什么 | 不保存什么 |
|---|---|---|
| `Evidence` | 原文、页码、章节、片段ID、哈希 | 当前患者是否触发、医生是否采纳 |
| `RecommendationStatement` | 推荐语句、动作、适用条件、禁忌条件、推荐等级、主证据 | 大段全文证据池 |
| 关系字段 | 连接方向、证据ID列表、来源集合 | 不替代 Evidence 原文 |

## 6. 正式迁移前必须 dry-run

后续若要迁移服务器数据，必须先生成 dry-run 报告：

```text
待迁移节点数
待迁移关系数
字段值变化数
可能冲突的 code/name/entityType
迁移前后硬闸门对比
回滚方案
```

未生成 dry-run 报告前，不得直接写 Neo4j。
