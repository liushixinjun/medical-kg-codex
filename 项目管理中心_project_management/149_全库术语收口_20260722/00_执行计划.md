# 全库术语与标准主数据收口执行计划

生成时间：2026-07-22

执行状态：8 个步骤已完成；服务器与本地验收通过。Oracle 既有业务字典未修改，第二批审核资产已生成。

## 目标

以 CDSS 有效标准字典作为业务身份候选，以教材、指南、药品监管资料和官方标准进行语义复核，对全库诊断、症状、体征、检查项目、检查发现、检验项目、检验细项、检验标本、药品、手术、治疗项目、生命体征、医学术语和别名完成一次性收口。

## 固定边界

1. 既有 Oracle 业务字典保持只读；新增、改名、合并、失效和删除只生成第二批审核清单。
2. Neo4j 写入前必须生成逻辑备份、变更增量、关系迁移清单、物理删除候选和回滚包。
3. 同义词仅表示同一概念；类别成员、剂型、品牌和类型改判分别使用明确关系。
4. 模型只做候选识别和查证线索，不作为医学来源。
5. 每项自动修改必须有确定规则或权威依据；无法唯一确定的项目进入审核队列。

## 一次性执行范围

1. 冻结服务器、Oracle审核队列和本地术语字典基线。
2. 建立全类型术语语义分类与自动化回归测试。
3. 扫描全库并生成自动修复、待查证、人工歧义三层清单。
4. 生成 Neo4j 修复增量、回滚包并通过写库前硬闸门。
5. 安全写入 Neo4j，迁移名称、别名、类型和关系。
6. 继续压缩 Oracle 候选，生成第二批中文审核页面。
7. 执行 AMI 与心肌病疑似诊断和 CDSS 调用验证。
8. 完成全库复核、报告、步骤记录、踩坑日志、版本与 Git 交付。

## 完成标准

- 图谱同名重复、无效简称主名、别名与类别成员混用、错误类型、孤立标准主数据均为 0。
- 所有自动改动能够从报告定位到原节点、目标节点、依据和回滚记录。
- 知识内容完整性、正式推荐链路、主数据质量、术语语义质量四类检查全部通过。
- AMI 与心肌病能够返回标准诊断、支持/排除条件、检查检验、治疗动作和证据来源。
- Oracle 既有业务字典写入仍为 0；待确认事项压缩为用户可批量审核的少量分组。
