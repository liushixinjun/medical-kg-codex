# 心血管内科月度建设计划（2026-07）

更新时间：2026-07-15 00:00:00

## 1. 本月最终目标

2026 年 7 月优先完成心血管内科以下疾病大类/专病图谱建设，并形成可供 CDSS 专病流程引擎使用的知识底座：

```text
冠心病
心肌病
急性心肌梗死
心力衰竭
高血压
心律失常
起搏治疗相关疾病
瓣膜病
肺动脉高压
```

## 2. 当前执行原则

| 原则 | 执行要求 |
|---|---|
| 双维度判定 | 每个疾病大类必须同时判断“教材骨架是否完成”和“指南/PDF正文解析是否完成” |
| 教材骨架优先归因 | 基础骨架缺口默认先视为《内科学》原文抽取不充分，必须先回原文补抽，不得直接归因于资料不足 |
| 当前基线优先 | 以 2026-07-09 后形成的 D6 教材骨架/服务器复核为当前基线 |
| 旧候选池不复用 | 2026-06-25 历史全书回捞候选池只作重抽取线索，不直接补图谱 |
| 回原文重抽取 | 每个疾病大类必须回到教材/PDF指南原文重新抽取 |
| 批次闭环 | 每批必须有来源清单、节点关系、证据、审计、导入包或 blocked 原因 |
| 证据链可用 | 诊断、检查、治疗、随访、禁忌、推荐动作必须绑定可追溯证据 |
| 写库受控 | 未通过 G2 不写 Neo4j；写库后必须服务器 postcheck |

## 2.1 完成状态判定口径

| 项目 | 可以说明什么 | 不能说明什么 |
|---|---|---|
| G0 预检 | 已确认候选教材、指南、PDF来源 | 不能说明 PDF 正文已解析 |
| 教材骨架入库 | 基础疾病知识结构已进入图谱 | 不能替代指南证据和推荐等级 |
| 教材骨架缺口 | 优先提示抽取遗漏，需要回《内科学》原文复抽 | 不能直接判定为资料不足 |
| 指南/PDF解析入库 | 指南证据、推荐、阈值、禁忌等已抽取并入库 | 不能替代教材基础骨架 |
| 大类基线验收 | 服务器现有图谱通过质量检查 | 不能说明该大类已完成新批次解析 |
| CDSS决策层升级 | 现有图谱具备部分动态推荐能力 | 不能说明教材和指南两维度均完整 |

只有同时满足“教材骨架完成 + 指南/PDF解析完成 + G1/G2审计通过 + Neo4j入库 + 服务器复测通过”，才能标记为“完成交付”。

## 3. 建设顺序与当前双维度状态

| 顺序 | 疾病大类/专病 | 教材骨架状态 | 指南/PDF解析状态 | 当前真实结论 | 优先槽位 |
|---:|---|---|---|---|---|
| 1 | 冠心病 | 已有骨架和精修 | 已有旧批次和精修，需按新规则复核 | 不是未做，但不能直接认定最终完成 | 诊断、鉴别、再灌注、抗栓、调脂、随访 |
| 2 | 心肌病 | 已有骨架和精修 | 已有旧批次和精修，需按新规则复核 | 不是未做，但需补强 | 定义、分型、临床表现、影像、遗传、治疗、猝死风险 |
| 3 | 急性心肌梗死 | STEMI 已精修 | AMI相关指南已有部分解析和精修 | 作为技术样板，不代表冠心病整体完成 | 诊断组件、STEMI/NSTEMI 分流、PCI/溶栓、用药、证据链 |
| 4 | 心力衰竭 | 已有骨架 | 已有完整批次和入库 | 已解析，需深度补强 | HFrEF/HFpEF、分期、药物、器械、随访、禁忌 |
| 5 | 高血压 | 已有骨架 | 已有完整批次和入库 | 已解析，需主数据归并 | 原发/继发、靶器官损害、风险分层、降压治疗、随访 |
| 6 | 心律失常 | 大类骨架已有 | 房颤、室上速/房扑、室性心律失常、缓慢性心律失常已解析；大类未全覆盖 | 部分解析 | 房颤、室上速、室性心律失常、缓慢性心律失常、传导阻滞 |
| 7 | 瓣膜病 | 教材章节已进入骨架审计 | 仅 G0 预检，未解析 PDF 正文 | 未完成正式解析 | 狭窄/反流、超声指标、手术/介入适应证、抗凝、随访 |
| 8 | 肺动脉高压 | 当前无独立疾病节点 | 未解析 | 从 0 启动 | 分型、筛查、右心导管、风险分层、靶向治疗 |
| 9 | 起搏治疗相关疾病 | 有相关实体但未形成独立批次 | 未解析 | 未完成正式解析 | 起搏适应证、禁忌、植入流程、并发症、随访 |

## 4. 每个批次必须交付

| 类别 | 文件/结果 |
|---|---|
| 启动 | batch_config、G0 预检报告、来源路径确认 |
| 来源 | PDF/教材来源清单、去重结果、OCR 阻断清单 |
| 数据 | nodes.jsonl、relations.jsonl、evidence.jsonl、source ledger |
| 审计 | G1/G2 审计矩阵、required 缺口、空壳节点、重复实体、证据链报告 |
| CDSS | RecommendationStatement、ClinicalRule、PathwayStage、诊断/鉴别/治疗/随访动作 |
| 入库 | 导入包、回滚清单、服务器 postcheck；如不写库，必须说明原因 |
| 交接 | 步骤记录、踩坑日志、错误指纹、当前进度与后续计划 |

## 5. 不再做的事

1. 不再拿历史全书回捞候选池直接补骨架。
2. 不再把旧候选池抽样问题当成当前骨架质量缺陷。
3. 不再绕过 G1/G2 直接写 Neo4j。
4. 不再只生成疾病级实体清单而缺 CDSS 推荐动作和证据链。
5. 不再让单个非阻断节点拖住整个疾病大类批次。
6. 不再把教材骨架缺口直接说成资料不足；必须先回《内科学》原文复抽。

## 6. 下一步唯一动作

先执行 `瓣膜病` 正式解析，因为它已有 G0 预检和 21 份候选资料；随后执行 `肺动脉高压` 从 0 建批次，再执行 `起搏治疗相关疾病`。冠心病、心肌病不再按“未做”处理，而是按新规则做回原文复核和 CDSS 决策链补强。
