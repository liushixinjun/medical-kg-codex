# 心血管内科月度建设计划（2026-07）

更新时间：2026-07-12 09:45:00

## 1. 本月最终目标

2026 年 7 月优先完成心血管内科以下疾病大类/专病图谱建设，并形成可供 CDSS 专病流程引擎使用的知识底座：

```text
心肌病
冠心病
急性心肌梗死
心力衰竭
心律失常
起搏治疗相关疾病
瓣膜病
肺动脉高压
```

## 2. 当前执行原则

| 原则 | 执行要求 |
|---|---|
| 当前基线优先 | 以 2026-07-09 后形成的 D6 教材骨架/服务器复核为当前基线 |
| 旧候选池不复用 | 2026-06-25 历史全书回捞候选池只作重抽取线索，不直接补图谱 |
| 回原文重抽取 | 每个疾病大类必须回到教材/PDF指南原文重新抽取 |
| 批次闭环 | 每批必须有来源清单、节点关系、证据、审计、导入包或 blocked 原因 |
| 证据链可用 | 诊断、检查、治疗、随访、禁忌、推荐动作必须绑定可追溯证据 |
| 写库受控 | 未通过 G2 不写 Neo4j；写库后必须服务器 postcheck |

## 3. 建设顺序

| 顺序 | 疾病大类/专病 | 本月目标 | 优先槽位 |
|---:|---|---|---|
| 1 | 冠心病 | 完成大类图谱与 CDSS 决策层升级 | 诊断、鉴别、再灌注、抗栓、调脂、随访 |
| 2 | 心肌病 | 完成大类图谱与 CDSS 决策层升级 | 定义、分型、临床表现、影像、遗传、治疗、猝死风险 |
| 3 | 急性心肌梗死 | 形成专病样板 | 诊断组件、STEMI/NSTEMI 分流、PCI/溶栓、用药、证据链 |
| 4 | 心力衰竭 | 完成大类图谱与 CDSS 决策层升级 | HFrEF/HFpEF、分期、药物、器械、随访、禁忌 |
| 5 | 心律失常 | 完成大类图谱 | 房颤、室上速、室性心律失常、缓慢性心律失常、传导阻滞 |
| 6 | 起搏治疗相关疾病 | 完成专题图谱 | 起搏适应证、禁忌、植入流程、并发症、随访 |
| 7 | 瓣膜病 | 完成大类图谱 | 狭窄/反流、超声指标、手术/介入适应证、抗凝、随访 |
| 8 | 肺动脉高压 | 完成大类/专病图谱 | 分型、筛查、右心导管、风险分层、靶向治疗 |

## 4. 每个批次必须交付

| 类别 | 文件/结果 |
|---|---|
| 启动 | batch_config、G0 预检报告、来源路径确认 |
| 来源 | PDF/教材来源清单、去重结果、OCR 阻断清单 |
| 数据 | nodes.jsonl、relations.jsonl、evidence.jsonl、source ledger |
| 审计 | G1/G2 审计矩阵、required 缺口、空壳节点、重复实体、证据链报告 |
| CDSS | RecommendationStatement、ClinicalRule、PathwayStage、诊断/鉴别/治疗/随访动作 |
| 入库 | 导入包、回滚清单、服务器 postcheck；如不写库，必须说明原因 |
| 交接 | 步骤记录、踩坑日志、错误指纹、当前进度与后续计划 |

## 5. 不再做的事

1. 不再拿历史全书回捞候选池直接补骨架。
2. 不再把旧候选池抽样问题当成当前骨架质量缺陷。
3. 不再绕过 G1/G2 直接写 Neo4j。
4. 不再只生成疾病级实体清单而缺 CDSS 推荐动作和证据链。
5. 不再让单个非阻断节点拖住整个疾病大类批次。

## 6. 下一步唯一动作

从 `冠心病` 批次开始，按当前 D6 基线回到教材和指南原文重新抽取高价值槽位，形成 G1/G2 审计闭环；通过后再受控写库并做服务器 postcheck。
