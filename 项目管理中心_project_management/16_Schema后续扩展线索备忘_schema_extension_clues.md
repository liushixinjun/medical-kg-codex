# Schema 后续扩展线索备忘

用途：记录暂不立即修改 Schema、但后续可能需要扩展的结构化线索。这里不是正式 Schema，不作为当前抽取和入库硬规则。

## 2026-07-12 21:56:53｜病理相关实体是否需要独立建模

### 当前结论

暂不修改正式 Schema。

当前 Schema 已支持：

| 中文概念 | 实体类型 | 当前用途 |
|---|---|---|
| 病因 | `Etiology` | 记录疾病发生原因 |
| 病理生理 | `Pathophysiology` | 记录发病机制、病理过程 |
| 检查 | `Exam` | 可承载病理检查、活检等检查方式 |
| 检查指标/发现 | `ExamIndicator` | 可承载检查发现、影像/病理结果等 |

### 后续触发条件

如果后续出现以下场景，再评估新增病理专用实体：

- 需要解析病理报告；
- 指南或教材大量出现组织学、活检、形态学改变；
- 需要区分“病理机制”和“病理发现”；
- 临床 CDSS 需要根据病理发现触发诊断、分型、风险分层或治疗推荐。

### 可能扩展方案

候选新增实体：

| 中文概念 | 候选实体类型 | 示例 |
|---|---|---|
| 病理发现 | `PathologyFinding` | 心肌细胞肥大、纤维化、炎症细胞浸润、凝固性坏死 |

候选关系：

| 起点 | 关系 | 终点 | 示例 |
|---|---|---|---|
| 疾病 | `has_pathology_finding` | 病理发现 | 肥厚型心肌病 -> 心肌细胞肥大 |
| 病理检查 | `exam_has_indicator` | 病理发现 | 心肌活检 -> 炎症细胞浸润 |

### 当前处理规则

在正式扩展前：

1. 发病机制、病理过程仍进入 `Pathophysiology`。
2. 病理检查、活检方式进入 `Exam`。
3. 具体检查发现可暂入 `ExamIndicator` 或作为 Evidence 原文保留。
4. 不为了单个疾病提前新增 Schema 实体，等多病种复用需求明确后统一升级。
