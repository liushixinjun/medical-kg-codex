# 心血管内科双维度建设状态校正（2026-07-15）

## 1. 校正原因

2026-07-14 的阶段性交付报告把“现有图谱大类基线验收”与“新批次 PDF 正文解析完成”混在了一起，容易造成计划误判。

从 2026-07-15 起，心血管内科建设进度必须同时按两个维度判断：

1. 教材骨架维度：是否已从《内科学》等基础教材抽取疾病概述、病因机制、临床表现、检查、诊断鉴别、治疗、预后随访等基础结构。
2. 指南/PDF维度：是否已从指南、共识、路径、PDF/文档正文中完成证据抽取、节点关系生成、审计、入库和服务器复测。

只做“大类基线验收”不等于完成 PDF 解析；只做“资料预检”也不等于完成 PDF 解析。

补充硬规则：教材骨架内容原本就应优先来自《内科学》。如果基础骨架槽位缺失，默认先判定为“教材原文抽取不充分”，必须回到《内科学》对应章节重新抽取和审计；不得直接归因为“本地资料不足”。只有在教材原文确实未覆盖该疾病或该槽位时，才允许标记为教材覆盖不足。

## 2. 状态判定标准

| 状态 | 判定含义 | 能否算完成 |
|---|---|---|
| 教材骨架已入库 | 已从教材抽取并写入图谱，且通过基础硬闸门 | 只代表基础骨架完成 |
| 教材骨架待补强 | 有章节或部分实体，但槽位不完整；优先视为抽取不充分，需要回教材原文补抽 | 不算完成 |
| 指南/PDF已解析入库 | 已有来源清单、正文解析、证据抽取、数据实例、审计、入库和复测 | 可计入该维度完成 |
| 指南/PDF部分解析 | 只覆盖部分专病或旧批次，未覆盖整个大类 | 不算大类完成 |
| 仅G0预检 | 只筛选候选资料，不解析正文 | 不算完成 |
| 仅大类基线验收 | 只检查服务器现有图谱 | 不算解析完成 |
| 计划中 | 尚未正式进入解析流程 | 不算完成 |

## 4. 后续计划硬规则

每个疾病大类计划必须同时写清：

1. 教材骨架是否完成。
2. 指南/PDF是否完成解析。
3. 是否已经生成节点、关系、证据。
4. 是否已通过 G1/G2 审计。
5. 是否已写入 Neo4j。
6. 是否已完成服务器复测。

只有以上全部满足，才能标记为“完成交付”。

教材骨架缺口处理顺序固定为：

```text
先回《内科学》原文章节复抽
→ 再查教材骨架审计矩阵是否漏抽
→ 再判断是否需要其他权威教材/指南补充
→ 最后才允许标记为来源覆盖不足
```

## 6. 禁止口径

1. 禁止把“大类基线验收通过”说成“PDF已解析完成”。
2. 禁止把“G0预检完成”说成“批次完成”。
3. 禁止把“已有服务器节点”说成“教材骨架+指南PDF双维度完成”。
4. 禁止把 AMI 样板混同为冠心病大类整体完成。
5. 禁止在未回查《内科学》原文前，把教材骨架缺口归因为资料不足。
